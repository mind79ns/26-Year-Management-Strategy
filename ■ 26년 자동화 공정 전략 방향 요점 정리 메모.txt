■ 26년 자동화 공정 전략 방향 요점 정리 메모

1. 손실 시간 가시화 → 즉시 액션 체계
기존 문제: 설비가 왜, 얼마나 멈췄는지 정확히 몰라서 대책도 애매함
현실적 접근: 손실 시간을 초 단위로 기록 → 순위화 → 집중 타격
구체적 실행:

수작업 손실 기록 → 간단한 앱 전환 → 세부 유실항목 적용 간단한 원터치 이력 저장 → LINE별 일/주/월 기록현황 및 분석활용

작업자가 앱 터치만 하면 자동 기록
- 주요 목적 : 일반 유실은 MES 집계로 된다고 판단하기 때문에 접근의 관점은 순간유실에 대한 세분화 항목 적용 부분이 되어야 한다.
- 방향성 : 운영은 매우 간단해야한다 / 작업자는 번거로운 업무가 추가 발생시 일시적 유지 활동은 가능하나 반복적인 업무증가 및 추가행위에 대한 지속유지가 실제 현장 운영상 어렵다는걸 직시하고 쉬운 운영 방침으로 계획을 잡고 시스템화 반영해야한다.
- 어떻게 할것인가 : 우선 모든 경우의 수 항목 LOSS,불량,고장,에러,지연상황 등을 나열하고 기본 DB로 활용 , / 각 공정 LINE 작업자는 로그인을 통해 위치(공정,LINE) 지정하고 단순 원터치로만(항목 누르면 분선택) 이력들이 저장되어야 한다.
- 활용도 : 누적된 각 정보 이력들을 기준으로 순간유실의 딥한 관리 품목 순위화 ,WORST, 집중타격 개선 운영 방침으로 현장 활동한다.
- 최종목표 : 기회손실을 줄이므로 인한 점당 가공비 직접 감소 효과 / 손실시간 5% 감소 개선 목표


2. 불량 사진 즉시 공유 → 재발 방지 강화
기존 문제: 불량 발생 → 구두 전달 → 일부 공유 및 재발발생 
현실적 접근: 사진 찍으면 자동으로 모든 조에 공유 + 간단한 조치사항 입력
구체적 실행:

불량 사진 공유 게시판 (사내로컬 개발 활용)

현장에서 스마트폰으로 사진 찍기 → 공정/설비 태그
조치 내용 한 줄 메모 (예: "에어압 2.5 → 3.0 조정")
전 조 자동 알람 → 재발 방지 (주요 이슈내용 및 전달사항 포함) 예시 주간조 이슈항목 → 야간조 자동 공유


월별 재발 불량 추적

같은 불량이 또 나오면 "재발" 자동 표시
재발 불량 Zero 목표 설정 → KPI화를 목표로 진행 


투자: 개인 개발 SYSTEM 적용 (회사로컬서버 및 모바일 사진업로드 기능)
효과: 재발 불량 30% 감소 → 품질불량 10% 목표 달성 가능

  • 병목 공정 표적 타격
  • 주간 손실 리포트 자동화 개선
